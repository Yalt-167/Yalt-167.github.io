Description

Multiplayer Networking:

This project is a multiplayer Tic Tac Toe game developed in C++ using SDL for rendering/input handling, WinSock2 for network communication, and ImGUI for the user interface and in-game chat system. The application allows two players to connect over a local network or the internet and play in real time through a client/server architecture.

Network synchronization is handled using TCP sockets through WinSock2 to ensure reliable transmission of game state updates and chat messages between players. SDL is responsible for window management, rendering, and input processing while ImGUI provides a lightweight and responsive debugging and chat interface. The project focuses on real-time communication, game state synchronization, and low-level networking concepts in C++.

How it works:

This project is a real-time multiplayer Tic Tac Toe game written in C++ using SFML for rendering and input handling, WinSock2 for networking, and ImGUI for the in-game interface and chat system. The application uses a client/server architecture where players connect to a central server that synchronizes the game state and relays chat messages between connected clients.

The architecture is separated into multiple layers to keep rendering, networking, packet serialization, and gameplay logic independent from one another. This structure makes the networking code easier to maintain and allows the server to validate game actions before broadcasting updates to clients. 


// Socket.hpp

class Socket
{
public:
    Socket();
    ~Socket();
protected:
    void InitWSA();

    void Cleanup() const;
public:
    virtual void Run() = 0;

protected:
    SOCKET socket_;
    WSADATA wsaData; // contains infos about WIndows sockets impl
    sockaddr_in serverAddr = sockaddr_in();
    // for specifying IPv4 socket addresses && protocols

    ::Packet header;
    char headerBuffer[sizeof(Packet)];
};

The Socket layer acts as a low-level abstraction over WinSock2 sockets. It centralizes connection handling, sending and receiving data, and resource cleanup. This avoids duplicating networking code between the client and server implementations. 


// PacketTypes.hpp

enum class PacketType
{
	Disconnection, // packet associated => none
	Play, // packet associated => int[1]
	// <int: slot>
	PlayResult, // packet associated => int[3]
	// {
	// <int: the play itself>
	// <int: wether u can play>,
	// <int: who played>
	// }
	ChatMessage, // packet associated => char[sizeSpecifiedInHeader]
	// <char*, the message>
	CatchupPacket, // packet associated => int[9]
	// basically the grid state if u got disconnected
};

// Packet.hpp

struct Packet
{
public:
	::PacketType PacketType;
	int Size;

	// returns self as char* (basically self but serialized)
	char* Set(const ::PacketType packetType, const  int size)
	{
		PacketType = packetType;
		Size = size;

		return reinterpret_cast<char*>(this);
	}

};


// packets constructed as such:
// send packets two by two

// header
// <what><size> -> <PacketHeader, int>
// could easily infer size from header if it wasn t for chat msgs

// then body
// <body> -> <char[<header.Size>]>

Communication between the server and clients is handled through packets. Each packet contains a small header describing the packet type and payload size, allowing the receiver to correctly deserialize incoming data. This packet-based approach makes the protocol scalable and easier to extend with new features.

void ServerSocket::Send(
    const char* data, const PacketType what, const int size,
    const PacketSendTarget target
    )
{
    if (target == PacketSendTarget::Broadcast)
    {
        for (auto& [__, socket] : clientSockets)
        {
            _ = send(socket, header.Set(what, size), sizeof(Packet), 0);

            _ = send(socket, data, size, 0);
        }
    }
    else
    {
        _ = send(
            clientSockets[static_cast<int>(target)],
            header.Set(what, size), sizeof(Packet),
            0
        );

        _ = send(
            clientSockets[static_cast<int>(target)],
            data, size,
            0
        );
    }
}

The entry point is Game::Run() which is overriden in both GameServer and GameClient. Which one you call actually defines your role. GameServer opens a connection for two clients and handle game flow from there. 


#define AwaitMilliseconds(ms) \
	std::this_thread::sleep_for( \
		std::chrono::milliseconds(ms) \
	)

void GameServer::Run()
{
    // this thread keeps listening for incoming connection
    // for the entire lifetime of the server
	connectionThread = new std::thread(&ServerSocket::Run, serverSocket);

    // await one connection to actually start a session
	while (serverSocket.GetConnectedClientsCount() < 1) // await one player
        // wait a bit to avoid putting unnecessary load on the CPU
        // otherwise it would compute
        // serverSocket.GetConnectedClientsCount() < 1
        // in a loop as fast as possible
		AwaitMilliseconds(500);

	std::cout << "Awaiting one more player to start the game..."
            << std::endl;

    // while at least one player is connected keep the session alive
	while (serverSocket.GetConnectedClientsCount() > 0)
	{
        // if awaiting a player
		if (serverSocket.GetConnectedClientsCount() < 2) 
		{
            // again: avoid frying CPU
			AwaitMilliseconds(500);
			continue;
		}

        // runs once everytime we reach two players

        // sends the grid to everyone (useful for reconnection)
		serverSocket.Send(
			Grid::Serialize(gridState),
			PacketType::CatchupPacket,
			sizeof(char) * 9,
			PacketSendTarget::Broadcast
		);

        // send to each client wether its their turn
		for (int i = 0; i < 2; i++) // loop over clients
		{
            // wether that client has the turn
			startupPacket[1] = static_cast<int>(i == playerTurn); 
			serverSocket.Send(
				reinterpret_cast<char*>(startupPacket),
				PacketType::PlayResult, sizeof(int) * 3,
				static_cast<PacketSendTarget>(i)
			);
		}

        // await a disconnection
		while (serverSocket.GetConnectedClientsCount() == 2)
            // still avoid toasting CPU
			AwaitMilliseconds(500); // await !players

		std::cout << "Awaiting another player to continue the game"
                << std::endl;
	}
    // no more player connected -> end session
	std::cout << "No more player connected. Shutting down the server" 
            << std::endl;
}

// here s how the server socket handles connections
void ServerSocket::Run()
{
    sockaddr_in clientAddr;
    int addrLen = sizeof(clientAddr);
    while (true)
    {
        while (connectedClients == 2)
            AwaitMilliseconds(500); // no roasting CPU

        // accept() is blocking and block execution
        // until it actually receives a connection to accept
        SOCKET clientSocket = accept(socket_, (sockaddr*)&clientAddr, &addrLen);
        
        // self explanitory
        if (clientSocket == INVALID_SOCKET)
        {
            std::cerr << "Failed to accept client" << std::endl;
            continue;
        }

        // if the connection failed and received invalid packet
        if (recv(clientSocket, headerBuffer, sizeof(Packet), 0) <= 0)
        {
            std::cerr << "Gone too soon" << std::endl;
            continue;
        }
        
        // receive the new player name
        // (packet with the size + packet with actual data)
        int nameSize = reinterpret_cast<int*>(headerBuffer)[1] + 1;
        char* nameBuffer = new char[nameSize];

        _ = recv(clientSocket, nameBuffer, nameSize, 0);
        
        // assign the first free index to the client
        int clientIndex = freeClientIDs[0];
        freeClientIDs.erase(freeClientIDs.begin());

        std::cout << nameBuffer << " connected" << std::endl;
        connectedClients++;

        // leave the connection handling to a thread
        // to avoid blocking execution here
        clientSockets[clientIndex] = clientSocket;
        clientThreads[clientIndex] = std::thread(
            &ServerSocket::HandleClient, this,
            clientSocket, nameBuffer, clientIndex
        );
        clientThreads[clientIndex].detach();
    }
}

void ServerSocket::HandleClient(
    SOCKET clientSocket, const char* name, const int clientNumber
    )
{
    // restore chat history
    // in case its a reconnection
    // if not the chat log are probably empty
    // unless the other player s been having a party by himself
    GameServer::RestoreChatMessages(clientNumber);

    std::string loginLog = "[Server]: " + std::string(name) + " connected";
    GameServer::StoreChatMessage(loginLog);
    Send(
        loginLog.c_str(),
        PacketType::ChatMessage,
        static_cast<int>(loginLog.size()) + 1,
        PacketSendTarget::Broadcast
    );

    char headerBuffer[sizeof(Packet)];
    bool socketOpen = true;
    while (socketOpen)
    {
        if (recv(clientSocket, headerBuffer, sizeof(Packet), 0) <= 0)
        {
            std::cout << "[" << name << "] disconnected" << std::endl;
            break;
        }

        Packet header = *reinterpret_cast<Packet*>(headerBuffer);

        if (GameServer::LogEveryPacket)
        {
            std::cout
                << "Got " << Packet::LegiblePacketType(header.PacketType)
                << " packet from [" << name
                << "] || Size: " << header.Size
                << std::endl;
        }

        // server socket forwards the packet to the appropriate method
        // (determined based on its header)
        switch (header.PacketType)
        {
            case PacketType::Disconnection:
                HandleDisconnection(clientSocket, name, clientNumber);
                socketOpen = false;
                break;

            case PacketType::ChatMessage:
                HandleChatMessage(clientSocket, clientNumber, header.Size);
                break;

            case PacketType::Play:
                HandlePlay(clientSocket, clientNumber);
                break;
        }
    }
    
    delete[] name;
    
    clientThreads.erase(clientNumber);
    clientSockets.erase(clientNumber);

    _ = closesocket(clientSocket);

    freeClientIDs.push_back(clientNumber);
    connectedClients--;
}

void GameClient::Run()
{
	ImGUI::SFML::Init(*window->RenderWindow);

	if (PickUsername())
		return;

	clientSocket = new ClientSocket(userName, &chatMessages);
	clientSocket->Run();


	while (shouldRun)
	{
		Play();
		Chat();
		Render();
	}
}

void GameClient::Play()
{
	const int play = GatherInput();

	if (play == QUIT)
	{
		clientSocket->Send(
			reinterpret_cast<const char*>(&QUIT),
			PacketType::Disconnection,
			sizeof(int)
		);

		shouldRun = false;
		return;
	}

	if (play != INVALID_PLAY && canPlay)
		clientSocket->Send(
            reinterpret_cast<const char*>(&play),
            PacketType::Play, sizeof(int)
        );
}

void GameClient::Chat()
{
	ImGUI::SFML::Update(*window->RenderWindow, sf::Time());

	ImGUI::Begin("Chat Window", nullptr, ImGuiWindowFlags_NoMove);
	ImGUI::SetWindowPos(ImVec2(620, 10));

	ImGui::BeginChild("ChatBox", ImVec2(250, 520), true);

	// Display all the chat messages
	for (const auto& message : chatMessages)
		ImGui::TextWrapped("%s", message.c_str());

	ImGui::EndChild();

	// Input box to enter new chat message
	static char inputText[256] = "";
	if (
        ImGui::InputText(
            "##ChatInput",
            inputText,
            IM_ARRAYSIZE(inputText),
            ImGuiInputTextFlags_EnterReturnsTrue)
        )
	{
		if (strlen(inputText) > 0)
		{
			std::string msg = 
                "[" + userName + "]: " + std::string(inputText);
			// Add the new message to the chat
			chatMessages.push_back(msg);
			clientSocket->Send(
                msg.c_str(),
                PacketType::ChatMessage,
                static_cast<int>(msg.length()) + 1
            );
		}
		// Clear the input field after sending the message
		inputText[0] = '\0';
	}

	ImGUI::End();
}

The server is authoritative and validates(or invalidates) every move received from clients. Once validated, updates are broadcast to all connected players to keep the board synchronized. The same system is also used for the in-game chat, ensuring reliable message delivery between clients. Separating the networking layer (sockets) from gameplay logic makes the project easier to extend into larger multiplayer games in the future. In this project GameClient wraps the client socket while GameServer wraps the server socket. The actual network code could be taken out of the project and used in another by just replacing the GameServer and GameClient which hold the logic. The ClientSocket and GameServer have no knowledge of (and therefore no dependency to) the game logic
